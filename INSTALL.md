# Инструкция по установке и настройке MTG AdminPanel

Этот документ поможет вам развернуть и настроить панель управления для MTG-прокси.

### Общая архитектура

Система состоит из трёх ключевых компонентов:

1.  **Панель управления (Backend + Frontend)**: Основной сервер, который предоставляет веб-интерфейс и API для управления.
2.  **Нода (Node)**: Любой удалённый сервер (VDS/VPS), на котором будут запускаться сами прокси.
3.  **Агент (Agent)**: Небольшой сервис, который рекомендуется установить на каждую ноду для ускорения и повышения надёжности управления.

---

### Шаг 1: Установка панели управления

Панель управления устанавливается как единое приложение с помощью Docker.

**Требования:**
*   Сервер (подойдёт даже самый простой VDS/VPS) с установленным Docker и Docker Compose.
*   Доступ к серверу по SSH.

**Инструкция:**

1.  **Подключитесь к вашему серверу по SSH.**

2.  **Создайте директорию для панели:**
    ```bash
    mkdir mtg-panel && cd mtg-panel
    ```

3.  **Создайте файл `docker-compose.yml`:**
    ```bash
    nano docker-compose.yml
    ```
    Вставьте в него следующее содержимое:
    ```yaml
    services:
      mtg-panel:
        build: . # Сборка из локальных файлов (рекомендуется)
        container_name: mtg-panel
        restart: unless-stopped
        ports:
          - "3000:3000" # Вы можете изменить порт 3000 на любой другой
        volumes:
          - ./data:/data
          - ./ssh_keys:/ssh_keys:ro
        environment:
          - PORT=3000
          - DATA_DIR=/data
          - AUTH_TOKEN=${AUTH_TOKEN}
          - AGENT_TOKEN=${AGENT_TOKEN}
    ```

4.  **Создайте файл `.env` и сгенерируйте для него безопасные токены.**

    Выполните следующие команды, чтобы сгенерировать два уникальных токена и записать их в файл `.env`:

    ```bash
    echo "AUTH_TOKEN=$(openssl rand -hex 32)" > .env
    echo "AGENT_TOKEN=$(openssl rand -hex 32)" >> .env
    ```

    После выполнения этих команд ваш файл `.env` будет выглядеть примерно так (со случайными значениями):

    ```
    # Токен для входа в панель.
    AUTH_TOKEN=a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2

    # Токен для связи панели с агентами.
    AGENT_TOKEN=f6e5d4c3b2a1f6e5d4c3b2a1f6e5d4c3b2a1f6e5d4c3b2a1f6e5d4c3b2a1f6e5
    ```
    > **Важно:** Эти токены являются вашими ключами доступа. Храните их в безопасности.

5.  **Запустите панель:**
    ```bash
    docker compose up -d
    ```

**Готово!** Теперь панель управления доступна по адресу `http://<IP_вашего_сервера>:3000`. Откройте его в браузере и используйте значение `AUTH_TOKEN` из вашего `.env` файла для входа.

### Шаг 2: Добавление ноды (удалённого сервера)

Теперь, когда панель работает, добавим в неё ваш первый сервер для прокси.

1.  **Войдите в веб-интерфейс панели.**
2.  Перейдите в раздел **"Ноды"** и нажмите **"Добавить ноду"**.
3.  **Заполните форму:**
    *   **Название**: Любое удобное имя (например, "Helsinki VPS").
    *   **Host / IP**: IP-адрес вашего удалённого сервера.
    *   **SSH Авторизация**:
        *   **Рекомендуемый способ (SSH ключ)**:
            1.  Нажмите **"SSH ключ"**.
            2.  Нажмите на иконку загрузки и выберите ваш **приватный** SSH-ключ (обычно `id_rsa`). Он будет безопасно загружен на сервер панели.
            3.  Убедитесь, что **публичный** ключ (`id_rsa.pub`) добавлен в `~/.ssh/authorized_keys` на удалённой ноде.
        *   **Альтернативный способ (Пароль)**:
            1.  Выберите **"Пароль"** и введите пароль от `root` (или другого пользователя, которого вы указали).
    *   **Остальные поля** можно оставить по умолчанию.

4.  **Нажмите "Добавить".**

---

### Шаг 3: Установка агента на ноду (Рекомендуется)

Агент делает управление пользователями практически мгновенным.

1.  **В интерфейсе панели** откройте настройки только что добавленной ноды.
2.  Найдите раздел **"MTG Agent"** и нажмите **"Установить"**.
3.  Скопируйте предложенную команду. Она будет выглядеть примерно так:
    ```bash
    mkdir -p /opt/mtg-agent && cd /opt/mtg-agent && curl -fsSL https://raw.githubusercontent.com/MaksimTMB/mtg-adminpanel/dev/mtg-agent/install-agent.sh | bash
    ```
4.  **Подключитесь к вашей ноде по SSH** и выполните эту команду.
    Скрипт автоматически скачает, настроит и запустит агента в Docker. Он будет использовать `AGENT_TOKEN`, который вы задали в `.env` файле панели.

5.  **Проверьте статус агента** в интерфейсе панели. Он должен стать "доступен".

---

### Шаг 4: Создание пользователей (прокси)

Теперь самое главное — создание прокси для пользователей.

1.  В интерфейсе панели перейдите на страницу **"Клиенты"** для вашей ноды.
2.  Нажмите **"Добавить"**.
3.  **Заполните форму:**
    *   **Имя**: Короткое имя латиницей (например, `ivan` или `user1`).
    *   **Заметка**: Любой комментарий для себя.
    *   **Истекает**: Дата, после которой прокси автоматически отключится.
    *   **Лимит трафика**: Лимит в ГБ.
4.  Нажмите **"Создать"**.

Пользователь будет создан мгновенно (если установлен агент) или в течение нескольких секунд (через SSH). Вы сразу получите готовую `tg://` ссылку, которую можно передать пользователю.

На этом всё! Ваша система полностью настроена.

### Опционально: Шаг 5 - Подключение домена (HTTPS)

Чтобы получить доступ к панели по безопасному домену (например, `https://panel.your-domain.com`), мы рекомендуем использовать веб-сервер Caddy, который автоматически настраивает HTTPS.

**Требования:**
*   Доменное имя, DNS A-запись которого указывает на IP-адрес вашего сервера.

**Инструкция:**

1.  **Создайте `Caddyfile`** в той же директории, где находится ваш `docker-compose.yml`:
    ```bash
    nano Caddyfile
    ```
    Вставьте в него следующую конфигурацию, **заменив домен и email на ваши**:
    ```caddy
    {
        email your-email@example.com
    }

    panel.your-domain.com {
        reverse_proxy mtg-panel:3000
    }
    ```

2.  **Обновите `docker-compose.yml`**, чтобы он включал сервис Caddy. Замените всё содержимое файла на это:
    ```yaml
    services:
      mtg-panel:
        build: .
        container_name: mtg-panel
        restart: unless-stopped
        volumes:
          - ./data:/data
          - ./ssh_keys:/ssh_keys:ro
        environment:
          - PORT=3000
          - DATA_DIR=/data
          - AUTH_TOKEN=${AUTH_TOKEN}
          - AGENT_TOKEN=${AGENT_TOKEN}

      caddy:
        image: caddy:2-alpine
        container_name: caddy
        restart: unless-stopped
        ports:
          - "80:80"
          - "443:443"
        volumes:
          - ./Caddyfile:/etc/caddy/Caddyfile
          - caddy_data:/data

    volumes:
      caddy_data:
    ```
    > **Обратите внимание:** Мы убрали `ports` из сервиса `mtg-panel`, так как теперь Caddy будет управлять доступом извне.

3.  **Перезапустите вашу панель** с новой конфигурацией:
    ```bash
    docker compose up -d
    ```

Caddy автоматически получит SSL-сертификат для вашего домена, и ваша панель будет доступна по адресу `https://panel.your-domain.com`.
